# Plan 03-02: Test Suite Audit and E2E Tests

## Objective

Ensure the full unit test suite passes cleanly (no real API calls, no import errors), then fill `test_e2e.py` with E2E tests guarded by `ATTIO_API_KEY`. This plan is independent of 03-01 — it touches only test files, not repl.py.

## Context

**Existing test files (all in `agent-harness/cli_anything/attio/tests/`):**
- `conftest.py` — shared fixtures (`runner`, `mock_client`, `mock_config`)
- `test_client.py` — AttioClient unit tests using `HTTPXMock` (pytest-httpx 0.36.0)
- `test_commands.py` — records command tests
- `test_formatter.py`, `test_config.py`, `test_exceptions.py` — utility tests
- `test_notes.py`, `test_tasks.py`, `test_comments.py`, `test_lists.py`, `test_entries.py`, `test_attributes.py`, `test_objects.py`, `test_files.py`, `test_meetings.py`, `test_webhooks.py`, `test_workspace.py` — Phase 2 module tests
- `test_e2e.py` — currently empty

**pytest-httpx compatibility:** pytest-httpx 0.36.0 + httpx 0.28.1 are confirmed compatible. The `HTTPXMock` fixture is the standard pattern — no `responses` or `unittest.mock` patches on httpx needed.

**E2E skip pattern:** Use `pytestmark` at module level OR a reusable `e2e` marker per class. The `pytestmark` approach is cleaner since the entire file is E2E.

**pyproject.toml testpaths:** Check `pyproject.toml` for `testpaths` setting — the E2E file must be in a discovered path.

**ATTIO_API_KEY source for E2E run:** The key lives in the local `.env` file at `C:\Users\mitch\Everything_CC\attio-cli\.env`. Load it with `python-dotenv` OR export the var before running pytest. E2E tests use `CliRunner` which invokes the real CLI — the CLI's own `load_config()` reads the env var, so no special setup is needed in test code beyond confirming the var is set.

### Task 3: Fill test_e2e.py

Replace the empty `agent-harness/cli_anything/attio/tests/test_e2e.py` with:

```python
"""E2E tests against the live Attio API (LeadGrow workspace).

These tests are skipped unless ATTIO_API_KEY is set in the environment.
To run: ATTIO_API_KEY=<key> pytest agent-harness/cli_anything/attio/tests/test_e2e.py -v

The CLI under test loads config via its own load_config() which reads ATTIO_API_KEY
from env or ~/.config/attio/config.toml — no special mock setup is needed here.
"""
import json
import os

import pytest
from click.testing import CliRunner

from cli_anything.attio.attio_cli import cli

pytestmark = pytest.mark.skipif(
    not os.getenv("ATTIO_API_KEY"),
    reason="ATTIO_API_KEY not set — skipping E2E tests",
)


@pytest.fixture
def runner() -> CliRunner:
    return CliRunner()


class TestE2EWorkspace:
    def test_workspace_self_returns_json(self, runner: CliRunner) -> None:
        """GET /v2/self — workspace identity endpoint returns valid JSON."""
        result = runner.invoke(cli, ["workspace", "self", "--json"])
        assert result.exit_code == 0, f"Unexpected exit: {result.output}"
        data = json.loads(result.output.strip())
        assert isinstance(data, dict), "Expected a JSON object"

    def test_workspace_members_returns_list(self, runner: CliRunner) -> None:
        """GET /v2/workspace/members — returns a JSON list."""
        result = runner.invoke(cli, ["workspace", "members", "--json"])
        assert result.exit_code == 0, f"Unexpected exit: {result.output}"
        data = json.loads(result.output.strip())
        assert isinstance(data, list), "Expected a JSON array"


class TestE2EPeople:
    def test_people_list_returns_records(self, runner: CliRunner) -> None:
        """GET /v2/objects/people/records — returns records list (limit 1)."""
        result = runner.invoke(cli, ["people", "list", "--json", "--limit", "1"])
        assert result.exit_code == 0, f"Unexpected exit: {result.output}"
        # Output may be a JSON array or newline-delimited JSON objects
        first_line = result.output.strip().split("\n")[0]
        data = json.loads(first_line)
        assert isinstance(data, (dict, list)), "Expected JSON output"


class TestE2ENotes:
    def test_notes_list_returns_json(self, runner: CliRunner) -> None:
        """GET /v2/notes — returns notes list."""
        result = runner.invoke(cli, ["notes", "list", "--json", "--limit", "5"])
        assert result.exit_code == 0, f"Unexpected exit: {result.output}"
        # Empty workspace is valid — just confirm exit code and valid JSON shape
        output = result.output.strip()
        if output:
            first_line = output.split("\n")[0]
            json.loads(first_line)  # Raises on invalid JSON


class TestE2ETasks:
    def test_tasks_list_returns_json(self, runner: CliRunner) -> None:
        """GET /v2/tasks — returns tasks list."""
        result = runner.invoke(cli, ["tasks", "list", "--json", "--limit", "5"])
        assert result.exit_code == 0, f"Unexpected exit: {result.output}"
        output = result.output.strip()
        if output:
            first_line = output.split("\n")[0]
            json.loads(first_line)  # Raises on invalid JSON
```
